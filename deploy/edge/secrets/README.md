# Edge enrollment secrets

Create these files outside version control and reference their absolute paths from the protected edge `.env`:

- `edge-bootstrap-token`: one-time, short-lived token created by `POST /api/cdn/nodes/:nodeId/bootstrap-token`.
- `edge-gateway-ca.crt`: CA chain used only for the first HTTPS bootstrap connection to the public edge gateway.

Use mode `0600` for the token and `0644` or stricter for the public CA. Remove the bootstrap token file after `edge-agent` reports healthy and the persistent `identity/identity.json`, client certificate, private key, and signing trust bundle exist in the `edge_data` volume.

The node private key is generated by the agent and remains in `edge_data`. Back up that volume under node-specific encryption. Never copy one edge identity volume to another node.
